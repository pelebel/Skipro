# Plan: Improve SkiPro Web App

This plan outlines the improvements for the SkiPro weather app to enhance its user experience, visual appeal, and technical structure.

## 1. UI/UX Improvements
- [ ] **Search Interaction**: Add an event listener to the search input to trigger `getWeather()` on "Enter" key press.
- [ ] **Loading State**: Implement a CSS loading spinner or skeleton loader to replace the current opacity-based loading.
- [ ] **Unit Toggle**: Add a toggle switch to switch between Celsius and Fahrenheit for temperature displays.
- [ ] **Weather Icons**: Map `weather_code` from the Open-Meteo API to appropriate weather icons.
- [ ] **Accessibility**: Add `aria-label` to the input and `type="submit"` to the button within a `<form>` for better accessibility.

## 2. Feature Enhancements
- [ ] **Location Persistence**: Implement `localStorage` to save and restore the last searched city.
- [ ] **Detailed Metrics**: Update `renderCard` to show wind speed and humidity alongside the Pro Score.
- [ ] **Extended Forecast**: Increase the loop for forecast cards from 5 to 7 days.
- [ ] **Improved Geolocation**: Add more robust error handling for `navigator.geolocation` (e.g., handling 'Permission Denied' vs 'Position Unavailable').

## 3. Code Refactoring
- [ ] **Modularization**: Split the single `index.html` into:
    - `index.html` (Structure)
    - `style.css` (Styling)
    - `script.js` (Logic)
- [ ] **State Management**: Implement a simple state object to manage `loading`, `error`, and `weatherData` states.
- [ ] **Code Cleanup**: Use more modern DOM manipulation patterns and improve variable naming.

## Verification Plan
- [ ] Verify that "Enter" key triggers search.
- [ ] Verify that the last searched city is remembered after page refresh.
- [ ] Verify that unit toggling correctly converts temperatures.
- [ ] Ensure the app remains responsive on mobile devices.
